{# Shared steps for the "Linking URLs to a " FAQ sections. The "Where:" pointer to the specific edit page lives in the surrounding section; only the flow itself is here. #}
  1. On the edit page, find the External links fieldset (above the form's bottom).
  2. Existing links list at the top — each has a Remove button.
  3. Under Add a new link:
    • Description — what the link says (e.g. "Wikipedia entry").
    • URL — the full link, including http:// / https://. Validated for shape on submit.
    • Type — dropdown of URL Types; optional. The type's icon (Font Awesome) renders next to the link in lists when there's no favicon.
    • Favicon path — leave blank and the server scrapes https://<host>/favicon.ico on submit, saves it under app/static/favicons/<host>_favicon.ico, and reuses that file for every other link from the same host. Set the field manually only when you want to override the auto-fetch (e.g. a custom path under static/).
  4. Add link. The new link appears at the top of the list, ready to be removed if you mis-typed.

Favicon auto-fetch is best-effort — if the host blocks bots, returns HTML instead of an image, or times out, the link is still saved but without a favicon. Set the path manually after the fact in that case.
